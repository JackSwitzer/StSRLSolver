# Parity audit: Watcher stances / scry / mantra / retain / ethereal

**Date:** 2026-04-21
**Auditor:** Opus 4.7 (subagent)
**Scope:** Watcher-specific combat mechanics — stances (Wrath/Calm/Divinity/Neutral), stance transitions + hooks, scry + Nirvana + Weave, mantra accumulation + Devotion, retain + Establishment/Perseverance/Windmill, ethereal end-of-turn, and Watcher-flavored powers (Mantra, Devotion, Establishment, Rushdown, Fasting, Study, BattleHymn, Nirvana, LikeWater, MasterReality, MentalFortress, Omega, WaveOfTheHand, FreeAttack, DeusExMachina, Vigor, Foresight, WrathNextTurn, EndTurnDeath, NoSkills, CannotChangeStance, LiveForever, BlockReturn, Mark, EnergyDown).
**Items reviewed:** 38 (4 stances, 6 stance hooks, scry/Nirvana/Weave/Foresight, mantra/Devotion/BattleHymn, 22 powers, 2 transient powers, retain pipeline incl. Establishment/Perseverance/Windmill/FlurryOfBlows, ethereal pipeline)
**Deviations found:** 14

## Summary

The owner-aware runtime cleanly handles Mental Fortress, Rushdown, Omega, LikeWater, Study, BattleHymn, Devotion (accrual), Divinity auto-exit, Wrath/Divinity damage multipliers, Calm/Divinity energy gains, FlurryOfBlows return, Nirvana block, Weave return, and the Perseverance/Windmill retain bonuses. Stance multipliers and the 10-mantra Divinity pivot match Java exactly.

However, a sizable cluster of Watcher-flavored powers and transient flags exists only as helper functions in `powers/buffs.rs` that populate `StartOfTurnResult` / `EndOfTurnResult` structs **which are never consumed by the engine**. `process_start_of_turn` and `process_end_of_turn` are defined but called only from unit tests. This silently drops CannotChangeStance gating, WrathNextTurn auto-enter, EndTurnDeath, LiveForever (AngelForm) block, EnergyDown (Fasting) energy drain at turn start, and NoSkills play-gating + end-of-turn removal. Several other deviations are timing/ordering bugs: Devotion triggers pre-draw instead of post-draw, Foresight skips its empty-deck shuffle, ScryAction omits Golden Eye (+2 scry), Nirvana/Weave fire after the scry pick rather than before/separately, and WaveOfTheHand clears at the next turn start rather than at end-of-round (so Weak still applies on enemy-turn block gains).

## Mechanics verified clean

- Stance multipliers: Wrath 2x outgoing/2x incoming, Divinity 3x outgoing, Divinity does **not** double incoming (`src/state.rs:21-65`, `src/damage.rs`). Matches `stances/WrathStance.java`, `stances/DivinityStance.java` exactly.
- Divinity auto-exit at start of turn (`src/engine.rs:1138-1141`). Matches `DivinityStance.java:atStartOfTurn`.
- Calm exit grants 2 energy (+1 on Violet Lotus) (`src/engine.rs:2959-2967`). Matches `CalmStance.java:onExitStance` + Violet Lotus relic.
- Divinity enter grants 3 energy (`src/engine.rs:2970-2972`). Matches `DivinityStance.java:onEnterStance`.
- Mantra accrual rule: ≥10 → enter Divinity, subtract 10, preserve remainder (`src/engine.rs:3002-3010`). Matches `MantraPower.java:stackPower`.
- Mental Fortress: any stance change → gain block (`powers/defs/stance.rs` `DEF_MENTAL_FORTRESS`). Matches `MentalFortressPower.java:onChangeStance`.
- Rushdown: entering Wrath → draw `amount` cards (`powers/defs/stance.rs` `DEF_RUSHDOWN` with `TriggerCondition::InStance(Wrath)`). Matches `RushdownPower.java:onChangeStance`.
- Omega: end-of-turn flat damage to all enemies, bypassing Strength/Vigor (routes through `deal_flat_damage`, `effects/interpreter.rs:1031-1066`). Matches `OmegaPower.java:atEndOfTurn` with `DamageType.THORNS`.
- LikeWater: end-of-turn block only when in Calm (`powers/defs/turn_end.rs` `DEF_LIKE_WATER` with `InStance(Calm)`). Matches `LikeWaterPower.java`.
- Study: end-of-turn add Insight to draw pile (`powers/defs/turn_end.rs` `DEF_STUDY`). Matches `StudyPower.java`.
- BattleHymn: start-of-turn add Smite to hand (`powers/defs/turn_start.rs` `DEF_BATTLE_HYMN`). Matches `BattleHymnPower.java`.
- Insight retain flag set via `runtime_meta.rs:121-122` (`retain: true` branch). Matches `Insight.java` (`selfRetain=true`).
- Insight exhaust flag set (`cards/watcher/insight.rs:8`). Matches `Insight.java` (`exhaust=true`).
- Retain/ethereal end-of-turn pipeline: ethereal cards exhaust, retain-tagged/flagged cards stay, rest discard. Runic Pyramid keeps the full hand except ethereal. Perseverance/Windmill on-retain hooks accumulate their bonuses. (`src/engine.rs:1359-1463`).
- Vigor: flat damage bonus on next Attack, consumed on use (`src/card_effects.rs:271-280`). Matches `VigorPower.java:atDamageGive + onUseCard`.
- Mantra card: `Pray` adds Mantra + queues Insight to draw pile (`cards/watcher/pray.rs`). Matches `cards/purple/Pray.java`.
- Prostrate: Mantra(2) + Block(4) at cost 0 (`cards/watcher/prostrate.rs`). Matches `cards/purple/Prostrate.java`.
- Master Reality upgrade on temp-card creation (`src/engine.rs:3015-3020`). Matches card paths creating Smite/Insight when Master Reality active.

## Follow-up questions

1. Are `process_start_of_turn` / `process_end_of_turn` (`powers/buffs.rs:620,704`) intentional scaffolding for a refactor that never landed, or should they be deleted outright to stop suggesting false coverage? (Relevant to D70's Equilibrium question too — same mechanism.)
2. Is the `StartOfTurnResult.enter_wrath` → WrathNextTurn path worth wiring given no current effect applies `WRATH_NEXT_TURN`? If we ship Simmering Fury+ or mod cards that rely on it, it becomes real. Suggest either deleting the helper or completing the hook.
3. For WS6 (onScry timing): is there an existing parity test for Nirvana+Foresight that locks the "block after pick" behavior, or can we safely flip to Java's pre-pick ordering? The only observable difference with current cards is Nirvana block sequencing vs the grid-select pause.
4. WS13: should we migrate Establishment to `CardInstance::modifyCostForCombat`-style ephemeral overrides to match Java exactly, or is the permanent-cost approach considered the canonical training-side simplification? (Affects card-transform + upgrade-in-combat interactions.)
5. For WS9 (AngelForm): does the training corpus actually draft DevaForm often enough to prioritize the PlatedArmor wiring, or can this be deferred? Same question applies to WS10/WS11 (NoSkills, Fasting) — which enemy movesets or cards need them active?
