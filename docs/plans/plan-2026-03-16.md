## Context

Training v5: 500K games/72h, avg 8.8, max F16, 0 wins. 1.7GB episode data collected. Need clean visibility into what happened and what's happening. Current dashboard has 33 components but is bloated — fresh rebuild focused on essentials. User is setting up RTX box in parallel.

## This Session: Dashboard Fresh Rebuild

**3 views, tab-switched:**
1. **Live** — Active training run (status, workers, top runs, system)
2. **Analysis** — Previous runs / progress audit (run comparison, floor curves, death analysis)
3. **Run Detail** — Deep dive into a single run or episode (combat replay, map, deck, Neow)

### Performance Stats Panel (in Live + Analysis views)

Track and display per-game and per-fight timing:
- **Time per rollout** (game): avg ms/game, P50/P95/P99 (counting parallel workers)
- **Time per fight** by room type: monster/elite/boss avg ms, turns, cards played
- **Boss fight detail**: MCTS sims used, search time ms, DFS vs MCTS fallback rate
- **Solver stats**: % decisions using DFS vs GumbelMCTS vs heuristic fallback
- **Inference latency**: avg batch size, queue wait ms, forward pass ms

**Data source:** Already partially in `recent_episodes.json` (`duration_ms` per combat). Need to add:
- `solver_type` field to combat log (which solver made each decision)
- `mcts_sims` and `search_ms` for MCTS decisions
- Aggregate timing in `status.json` (avg_game_ms, avg_fight_ms by type)
